# E30 Acceptance

- Без `X-Tenant-Id` → 401 (или код из конфигурации).
- Операции assign/unassign влияют только на данные текущего тенанта; в shipped runtime связи хранятся в DB и фильтруются по tenant_id.
- Превышение квоты max_assignments → 429 {code:"quota_exceeded"}; логика мягкая — разрешается повтор после уменьшения нагрузки/очистки.
- В DB-режиме миграции применены; запросы фильтруются по tenant_id.
