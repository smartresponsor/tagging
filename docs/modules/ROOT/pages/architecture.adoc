== Current architectural reading

The shipped runtime is centered on `host-minimal/` with route truth centralized in `tag.yaml`.
The repository follows the current Symfony-oriented `App\Tagging\` layout and keeps the canonical Tag branches under:

* `src/Application/Write/Tag/...`
* `src/Http/Api/Tag/...`
* `src/Infrastructure/Persistence/Tag/...`
* `src/Infrastructure/ReadModel/Tag/...`
* `src/Service/Core/Tag/...`

== Architectural boundary notes

* `README.md` remains the repository-facing overview.
* `repo-map.md` remains the structural inventory.
* hand-written architecture narratives remain in `docs/architecture/` and `docs/http/`.
* generated/reference artifacts stay outside the hand-written architecture narrative tree.

== What this Antora page does

* provides a stable architecture entry point for a central Antora aggregator
* points to the canonical narrative sources already present in the repository
* avoids copying large Markdown architecture documents into a second parallel tree
